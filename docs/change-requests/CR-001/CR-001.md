# CR-001: 支持自定义 URL + API Key 作为 AI Provider

## 需求描述

支持用户通过自定义 Base URL 和 API Key 接入任意兼容 OpenAI Chat Completions API 的 AI 服务商（如 OpenAI、Azure OpenAI、本地 Ollama、LM Studio、第三方代理等）。用户在设置页配置 Base URL、API Key 和规划/生成模型名称后，即可将该自定义端点作为 AI Provider 使用，替代内置的 Claude API Provider。

## 背景与动机

当前 M-04 AI Provider 通信层的 MVP 实现仅内置 `ClaudeAPIProvider`（通过 `@anthropic-ai/sdk` 对接 Anthropic Claude API），后续规划的第二个 Provider 是本地 OpenClaw（P2）。但存在以下实际需求：

1. **多模型选择**：用户可能希望使用 OpenAI GPT 系列、Azure OpenAI 企业端点、Gemini（via OpenAI-compatible API）等模型，不应被锁定在 Anthropic 生态。
2. **私有化部署**：企业用户可能有内网 AI 服务（如部署在私有云的 LLM 代理网关），需要自定义 Base URL 指向内网地址。
3. **本地模型**：开发者可能在本地运行 Ollama、LM Studio 等工具，这些工具均暴露 OpenAI-compatible API（默认 `http://localhost:11434/v1`），可免费本地推理。
4. **降低依赖门槛**：不要求用户必须有 Anthropic 账号，降低产品使用门槛。

这与 `docs/idea.md` 中"可插拔 AI 后端"的核心设计理念高度一致，是对该理念的自然延伸。

### 关键技术影响（需在 CR-4 详细设计）

1. **Prompt 格式适配**：Claude API 使用 Anthropic messages 格式（`role: "user" | "assistant"`，`content: ContentBlock[]`），OpenAI Chat Completions 使用标准格式（`role: "system" | "user" | "assistant"`，`content: string`）。`CustomOpenAIProvider` 需将 IntentOS 内部 prompt 构建逻辑转换为 OpenAI 格式。

2. **工具调用协议差异**：代码生成阶段（`generateCode`）当前依赖 `@anthropic-ai/claude-agent-sdk` 的 tool_use 协议；OpenAI-compatible API 使用 function calling 协议，格式不同。`CustomOpenAIProvider` 需实现独立的工具调用循环（不复用 claude-agent-sdk），影响 M-04 代码生成的实现复杂度。

3. **流式响应格式**：OpenAI SSE 格式（`data: {"choices":[{"delta":...}]}`）与 Claude SSE 格式（`data: {"type":"content_block_delta",...}`）不同，需在 `CustomOpenAIProvider` 内部统一映射为 IntentOS `PlanChunk` / `GenProgressChunk` 格式。

4. **M-05 生成器潜在影响**：`M-05 buildPlanSystemPrompt()` 和 `buildGeneratePrompt()` 构建的 prompt 内容目前针对 Claude 优化（包含 Claude 特有的 thinking 引导词等）。切换到自定义 Provider 时，可能需要 Provider 感知的 prompt 模板（或至少移除 Claude 专有引导词），M-05 或 M-04 需提供 prompt 适配机制。

### 变更范围评估

- **规模**: Large
- **是否引入新模块**: 否（在 M-04 内新增 `CustomOpenAIProvider` 实现类，无需新模块）
- **是否修改已有接口**: 是，具体接口变更：
  - `ProviderConfig`（`shared-types.md`、`interfaces.md`、`ai-provider-spec.md`、`m04-ai-provider.md`）
  - `AIProviderManager.setProvider()` 的 providerId 参数范围
  - IPC channel `ai-provider:set-provider` 的参数枚举
  - `APIKeyStore` 存储 key 的命名规范（新增 `custom` 条目）
  - 新增 IPC channel：`settings:get-custom-provider-config` / `settings:set-custom-provider-config`
