## 1. SkillApp 架构概述

### 1.1 SkillApp 定义与运行模式

**SkillApp** 是由 AI Provider（M-05 SkillApp 生成器）自动生成的独立 Electron 应用。每个 SkillApp：
- 是完整的、可直接运行的 Electron 应用
- 拥有独立的主进程和渲染进程（进程隔离）
- 与 IntentOS Desktop 通过 Unix Socket（macOS/Linux）或 Named Pipe（Windows）进行 IPC 通信（非 BrowserWindow 嵌入）
- 内嵌 M-06 SkillApp 运行时模块，用于调用 Skill、访问系统资源、处理热更新

### 1.2 架构约束（必须遵守）

SkillApp 必须满足以下安全和隔离约束：

| 约束项 | 要求 | 说明 |
|--------|------|------|
| **进程隔离** | 独立 Electron 进程 | 与 Desktop 和其他 SkillApp 完全隔离，崩溃不影响其他应用 |
| **contextIsolation** | **必须启用**（true） | 渲染进程 JavaScript 隔离，防止 XSS 漏洞获取 Node.js 权限 |
| **nodeIntegration** | **必须禁用**（false） | 渲染进程禁止直接调用 Node.js API |
| **sandbox** | **必须启用**（true） | 渲染进程运行在 Chromium 沙箱中，额外限制 OS 资源访问 |
| **preload** | 必须使用绝对路径 | preload 脚本通过 contextBridge 暴露 API，是渲染进程唯一的外界接口 |
| **资源访问路径** | 只能通过 `window.intentOS` API | 禁止渲染进程直接调用 Node.js API、访问文件系统、网络等 |

## 2. 模板目录结构

M-05 生成器产出的 SkillApp 遵循以下固定目录结构：

```
src/skillapp-template/
├── package.json.template              # npm 依赖和应用元数据模板
├── main.js.template                   # Electron 主进程模板
├── preload.js.template                # preload 脚本模板（暴露运行时 API）
└── src/
    └── renderer/
        └── router.template.ts         # React Router 7 路由配置模板
```

代码生成器（M-05）填充这些模板后，在以下位置生成完整的 SkillApp 应用：

```
userData/apps/{appId}/
├── package.json                       # 填充后的 package.json
├── main.js                            # 填充后的 main.js
├── preload.js                         # 填充后的 preload.js
├── src/
│   ├── index.html                     # 渲染进程入口
│   ├── app/
│   │   ├── App.tsx                    # AI 生成的业务代码（应用根组件）
│   │   ├── pages/                     # AI 生成的页面组件
│   │   ├── components/                # AI 生成的 UI 组件
│   │   └── services/                  # AI 生成的业务逻辑

├── dist/                              # 构建产物
├── node_modules/                      # 依赖

    ├── state.json                     # 运行时状态
    ├── permissions.json               # 已授权权限
    ├── updates/                       # 热更新缓存
```

## 4. main.js.template 完整规范

```javascript
/**
 * SkillApp Electron 主进程入口
 * 由 M-05 SkillApp 生成器自动填充并生成
 */

const { app, BrowserWindow, ipcMain } = require('electron');
const path = require('path');

// 导入 SkillApp Runtime（M-06）
const runtime = require('@intent-os/skillapp-runtime/main');

// ============================================================================
// 1. 环境变量读取
// ============================================================================
// 这些环境变量由 IntentOS Desktop 在启动 SkillApp 时设置
const APP_ID = process.env.INTENTOS_APP_ID || '{{APP_ID}}';
const IPC_SOCKET_PATH = process.env.INTENTOS_SOCKET_PATH;
const DESKTOP_PID = process.env.INTENTOS_DESKTOP_PID;

let mainWindow;

// ============================================================================
// 2. 应用生命周期处理
// ============================================================================

app.whenReady().then(async () => {
  try {
    // ===== Step 1: 初始化 SkillApp Runtime（M-06）=====
    // Runtime 负责：
    // - 建立与 IntentOS Desktop 的 IPC 连接
    // - 完成握手验证
    // - 加载权限缓存
    // - 检查待应用的热更新
    await runtime.initialize({
      appId: APP_ID,
      appPath: __dirname,
      ipcSocketPath: IPC_SOCKET_PATH,
      desktopPid: DESKTOP_PID,
    });

    // ===== Step 2: 检查并应用待处理的热更新 =====
    // 如果上次更新被中断或应用未启动，此时补偿应用
    await runtime.applyPendingUpdates();

    // ===== Step 3: 创建 BrowserWindow =====
    // 安全配置：contextIsolation + sandbox + nodeIntegration:false
    mainWindow = new BrowserWindow({
      width: 1024,
      height: 768,
      show: false,  // 原地变形模式下先隐藏，由 Desktop 控制显示时机
      webPreferences: {
        // === 安全配置（三项必须全开） ===
        contextIsolation: true,     // 必须启用：隔离渲染进程 JavaScript
        nodeIntegration: false,     // 必须禁用：禁止直接调用 Node.js API
        sandbox: true,              // 必须启用：Chromium 沙箱隔离

        // === preload 脚本（绝对路径，必须指定） ===
        preload: path.join(__dirname, 'preload.js'),

        // === 其他安全配置 ===
        enableRemoteModule: false,
        webSecurity: true,
        allowRunningInsecureContent: false,
        nodeIntegrationInWorker: false,
        nodeIntegrationInSubFrames: false,
      },
    });

    // ===== Step 4: 导航安全限制 =====
    // 仅允许加载本地 SkillApp 文件，禁止导航到外部 URL
    mainWindow.webContents.on('will-navigate', (event, url) => {
      // 仅允许本地 file:// 协议
      const parsedUrl = new URL(url);
      if (parsedUrl.protocol !== 'file:') {
        event.preventDefault();
        console.warn(`[Security] Blocked navigation to: ${url}`);
      }
    });

    // 禁止打开新窗口（防止弹窗攻击）
    mainWindow.webContents.setWindowOpenHandler(({ url }) => {
      if (url.startsWith('http')) {
        // 外部链接用系统浏览器打开
        require('electron').shell.openExternal(url);
      }
      return { action: 'deny' };
    });

    // ===== Step 5: 加载应用页面 =====
    // 加载编译产物中的 index.html
    mainWindow.loadFile(path.join(__dirname, 'dist', 'index.html'));

    // 若为开发模式，打开 DevTools（可选）
    if (process.env.NODE_ENV === 'development') {
      mainWindow.webContents.openDevTools();
    }

    // ===== Step 6: 注册热更新处理器 =====
    // 监听来自 IntentOS Desktop 的热更新推送
    runtime.onHotUpdate(async (updatePackage) => {
      try {
        // Runtime 执行动态 import() 替换
        await runtime.applyHotUpdate(updatePackage, mainWindow.webContents);
        console.log(`[Runtime] Hot update applied: v${updatePackage.toVersion}`);
      } catch (error) {
        console.error(`[Runtime] Hot update failed, will fallback:`, error);
        // Runtime 会自动降级为 webContents.reloadIgnoringCache()
      }
    });

    // ===== Step 7: 报告应用就绪状态 =====
    // 通知 IntentOS Desktop 该 SkillApp 已启动完成
    // 此时 Desktop 可能继续执行「原地变形」动画
    runtime.reportStatus('running');

  } catch (error) {
    console.error('[Runtime] Initialization failed:', error);
    // 初始化失败，退出进程
    app.quit();
  }
});

// ===== 窗口关闭事件 =====
mainWindow.on('closed', () => {
  mainWindow = null;
});

// ===== 应用全部窗口关闭 =====
app.on('window-all-closed', () => {
  // 通知 IntentOS Desktop 应用正在停止
  runtime.reportStatus('stopping');

  // 清理运行时资源（IPC 连接、事件监听等）
  runtime.cleanup();

  // macOS 惯例：保持应用在后台（除非用户明确退出）
  // Windows/Linux 立即退出
  if (process.platform !== 'darwin') {
    app.quit();
  }
});

// ===== macOS 重新激活处理 =====
app.on('activate', () => {
  if (mainWindow === null) {
    // 若窗口被关闭但应用仍在，则重新创建窗口
    // （此处代码假设窗口生命周期管理由 IntentOS Desktop 控制，
    // 在正常使用中不会触发此分支）
  }
});

// ============================================================================
// 3. IPC Handler 注册（来自 preload.js / Runtime 的请求处理）
// ============================================================================

ipcMain.handle('runtime:callSkill', async (event, params) => {
  // Runtime 主进程模块通过此 IPC 请求调用 Skill
  // 实际处理由 Runtime 内部完成，这里仅作占位
  return await runtime.handleSkillCall(params);
});

ipcMain.handle('runtime:accessResource', async (event, params) => {
  // Runtime 主进程模块处理资源访问请求
  return await runtime.handleResourceAccess(params);
});

ipcMain.handle('runtime:requestPermission', async (event, params) => {
  // Runtime 主进程模块处理权限请求
  return await runtime.handlePermissionRequest(params);
});
```

### 关键环节说明

#### 环境变量说明

| 环境变量 | 格式 | 由谁设置 | 说明 |
|---------|------|---------|------|
| `INTENTOS_APP_ID` | 小写字母+数字+连字符 | IntentOS Desktop（M-03 生命周期管理器） | SkillApp 的唯一标识，在 Desktop 启动时通过 `child_process.spawn` 的 `env` 传入 |
| `INTENTOS_SOCKET_PATH` | Unix 路径或 Windows 管道路径 | IntentOS Desktop | IPC 通信的 Unix Socket（macOS/Linux）或 Named Pipe（Windows）路径 |
| `INTENTOS_DESKTOP_PID` | 数字 | IntentOS Desktop | Desktop 进程 ID，用于崩溃后验证 Desktop 是否仍活跃 |

#### 安全配置三要素（强制要求）

必须同时配置以下三项才能达到 Electron 安全基线：

```javascript
webPreferences: {
  contextIsolation: true,    // 1. 隔离渲染进程 JavaScript 执行上下文
  nodeIntegration: false,    // 2. 禁用渲染进程直接访问 Node.js API
  sandbox: true,             // 3. 启用 Chromium 沙箱
  preload: path.join(__dirname, 'preload.js'),  // 4. preload 脚本（绝对路径）
}
```

缺少任何一项都会削弱安全性。

#### Runtime 初始化流程

1. **读取环境变量** — 获取 APP_ID、SOCKET_PATH 等
2. **初始化 Runtime** — Runtime 建立 Unix Socket / Named Pipe 连接至 IntentOS Desktop，完成握手
3. **应用待处理更新** — 若有未完成的热更新，此时补偿应用
4. **创建 BrowserWindow** — 以严格的安全配置创建窗口
5. **加载页面** — loadFile 加载 dist/index.html
6. **注册热更新回调** — 监听来自 Desktop 的热更新推送
7. **报告就绪状态** — 通知 Desktop 应用启动完成

## 5. preload.js.template 完整规范

```javascript
/**
 * SkillApp preload 脚本
 * 通过 contextBridge 暴露 window.intentOS API 给渲染进程
 * 由 M-05 SkillApp 生成器自动生成
 */

const { contextBridge, ipcRenderer } = require('electron');

// ============================================================================
// 1. 通过 contextBridge 暴露安全 API
// ============================================================================
// 注意：这是渲染进程与主进程/Runtime 交互的唯一合法通道。
// 渲染进程（AI 生成的业务代码）通过 window.intentOS 调用这些 API。

contextBridge.exposeInMainWorld('intentOS', {

  // =========================================================================
  // Skill 调用接口
  // =========================================================================

  /**
   * 调用一个 Skill 执行某个方法
   * @param {string} skillId - Skill 的唯一标识（如 'data-cleaner'）
   * @param {string} method - 要执行的方法名（如 'clean'）
   * @param {object} params - 方法参数（JSON 可序列化对象）
   * @returns {Promise<any>} - Skill 执行结果
   * @throws {Error} - Skill 不存在、权限不足、执行超时等错误
   *
   * 使用示例：
   * const result = await window.intentOS.callSkill(
   *   'csv-cleaner',
   *   'clean',
   *   { filePath: '/path/to/data.csv', dedup: true }
   * );
   */
  callSkill: (skillId, method, params) =>
    ipcRenderer.invoke('runtime:callSkill', { skillId, method, params }),

  // =========================================================================
  // 资源访问接口（文件、网络、进程等）
  // =========================================================================

  /**
   * 通过 MCP 访问宿主 OS 资源
   * @param {object} request - 资源请求对象
   *   - {string} type - 资源类型（'fs' | 'net' | 'process'）
   *   - {string} action - 操作动作（取决于 type）
   *   - {object} params - 操作参数
   * @returns {Promise<any>} - 资源操作结果
   *
   * 使用示例（文件系统）：
   * const content = await window.intentOS.accessResource({
   *   type: 'fs',
   *   action: 'read',
   *   params: { path: '/path/to/file.txt' }
   * });
   *
   * 使用示例（网络）：
   * const response = await window.intentOS.accessResource({
   *   type: 'net',
   *   action: 'fetch',
   *   params: { url: 'https://example.com/api', method: 'GET' }
   * });
   */
  accessResource: (request) =>
    ipcRenderer.invoke('runtime:accessResource', request),

  // =========================================================================
  // 权限请求接口
  // =========================================================================

  /**
   * 请求用户授予访问特定资源的权限
   * 首次访问敏感资源（如写入文件、网络访问）时，会弹出授权弹窗
   * @param {object} request - 权限请求对象
   *   - {string} resource - 资源类型（如 'fs.write'）
   *   - {string} action - 具体操作
   *   - {object} params - 请求参数（如文件路径）
   * @returns {Promise<boolean>} - 用户是否授予权限
   *
   * 使用示例：
   * const allowed = await window.intentOS.requestPermission({
   *   resource: 'fs',
   *   action: 'write',
   *   params: { path: '/Users/user/Documents/output.csv' }
   * });
   * if (allowed) {
   *   // 执行写入操作
   * } else {
   *   // 权限被拒绝
   * }
   */
  requestPermission: (request) =>
    ipcRenderer.invoke('runtime:requestPermission', request),

  // =========================================================================
  // 应用信息接口
  // =========================================================================

  /**
   * 获取当前 SkillApp 的应用信息
   * @returns {Promise<object>} - 应用元数据
   *   - {string} appId - 应用 ID
   *   - {string} appName - 应用名称
   *   - {string} version - 应用版本
   *   - {array} skillIds - 该应用使用的 Skill 列表
   *   - {array} permissions - 应用声明的权限列表
   */
  getAppInfo: () =>
    ipcRenderer.invoke('runtime:getAppInfo'),

  // =========================================================================
  // 事件监听接口
  // =========================================================================

  /**
   * 监听 Skill 执行错误事件
   * 当某个 Skill 调用失败时触发此事件
   * @param {function} callback - 回调函数，接收错误信息对象
   * @returns {function} - 用于取消监听的函数
   *
   * 使用示例：
   * const unsubscribe = window.intentOS.onSkillError((error) => {
   *   console.error(`Skill ${error.skillId} failed:`, error.message);
   *   // 向用户展示错误提示
   * });
   *
   * // 稍后取消监听
   * unsubscribe();
   */
  onSkillError: (callback) => {
    const handler = (_event, data) => callback(data);
    ipcRenderer.on('runtime:skillError', handler);
    // 返回取消监听函数
    return () => ipcRenderer.removeListener('runtime:skillError', handler);
  },

  /**
   * 监听热更新事件
   * 当 IntentOS Desktop 推送新版本更新时触发
   * @param {function} callback - 回调函数
   * @returns {function} - 用于取消监听的函数
   *
   * 使用示例：
   * window.intentOS.onHotUpdate((update) => {
   *   console.log(`Update available: v${update.toVersion}`);
   *   // 提示用户应用已更新
   * });
   */
  onHotUpdate: (callback) => {
    const handler = (_event, data) => callback(data);
    ipcRenderer.on('runtime:hotUpdate', handler);
    return () => ipcRenderer.removeListener('runtime:hotUpdate', handler);
  },
});
```

### API 设计原则

1. **所有 API 都是异步的**（返回 Promise）
   - 原因：跨越多个进程边界，任何环节都可能有延迟
   - 业务代码必须使用 `async/await` 或 `.then()`

2. **IPC Channel 名称固定**（不由 AI 生成器修改）
   - `runtime:callSkill` — Skill 调用
   - `runtime:accessResource` — 资源访问
   - `runtime:requestPermission` — 权限请求
   - `runtime:getAppInfo` — 应用信息
   - `runtime:skillError` — 错误事件
   - `runtime:hotUpdate` — 热更新事件

3. **所有参数和返回值都是 JSON 可序列化的**
   - 不支持传递函数、Date 对象等

```typescript
//   index?: boolean,                 // 是否为默认路由
// }

const routes = [
  {
    path: '/',
    element: <AppLayout />,
    children: [
      {
        index: true,

      {

        ),
      },
      {
        path: '/config',
        element: (

        ),
      },
      {

        element: (

      },
    ],
  },

  // 使用 hash 路由（#）以适应 file:// 协议加载
  future: {

// ============================================================================
```

### 路由配置格式规范

代码生成器在填充 `{{ROUTES}}` 时，必须遵守以下格式：

```typescript
// 每个新增路由都遵循这个结构

  element: (

      <PageNameComponent />

```

**关键约束**：

1. **路径必须以 `/` 开头** — 如 `/import`, `/config`，不支持相对路径
2. **每个路由必须包裹在 `<Suspense>` 中** — 处理动态加载的加载状态
3. **不允许嵌套路由** — 所有路由都是 AppLayout 的直接子路由
4. **组件导入使用动态 `React.lazy()`** — 不使用静态 `import`，以支持热更新

## 8. SkillApp 业务代码规范

### 8.1 Skill 调用规范

**仅有合法途径**：通过 `window.intentOS.callSkill()` 调用 Skill

```typescript
// ✅ 正确做法
async function cleanData(filePath: string) {
  try {
    const result = await window.intentOS.callSkill(
      'data-cleaner',
      'clean',
      { filePath, options: { dedup: true } }
    );
    return result;
  } catch (error) {
    console.error('Skill 调用失败:', error);
  }
}

// ❌ 禁止：直接 import/require Skill
import DataCleaner from '@intent-os/skill-data-cleaner'; // 禁止！
const cleaner = new DataCleaner(); // 禁止！

// ❌ 禁止：动态 require
const skill = require(`@intent-os/skill-${skillName}`); // 禁止！
```

### 8.2 资源访问规范

**仅有合法途径**：通过 `window.intentOS.accessResource()` 访问系统资源

```typescript
// ✅ 正确做法：读取文件
const fileContent = await window.intentOS.accessResource({
  type: 'fs',
  action: 'read',
  params: { path: '/path/to/file.csv' }
});

// ✅ 正确做法：发起网络请求
const response = await window.intentOS.accessResource({
  type: 'net',
  action: 'fetch',
  params: {
    url: 'https://api.example.com/data',
    method: 'GET',
    headers: { 'Authorization': 'Bearer token' }
  }
});

// ❌ 禁止：直接调用 Node.js fs 模块
import fs from 'fs'; // 禁止！contextIsolation: true 会阻止此操作

// ❌ 禁止：直接使用 fetch（不经过权限控制）
fetch('https://api.example.com/data'); // 禁止！应使用 window.intentOS.accessResource

// ❌ 禁止：使用 child_process 启动子进程
const { execSync } = require('child_process'); // 禁止！
execSync('some-command'); // 禁止！
```

### 8.3 禁止的危险操作

以下操作在 SkillApp 中 **完全禁止**，代码生成器必须避免生成此类代码：

```typescript
// ❌ 禁止：eval() 和 Function() 动态执行代码
eval('dangerous code'); // 禁止！
new Function('return ...')(); // 禁止！
setTimeout('eval(something)', 100); // 禁止！

// ❌ 禁止：使用 innerHTML 渲染未转义的用户输入
element.innerHTML = userInput; // 禁止！改用 textContent 或 React 自动转义

// ❌ 禁止：访问全局对象进行权限提升
window.process; // undefined（contextIsolation 已隔离）
window.require; // undefined（nodeIntegration: false）

// ❌ 禁止：利用原型链进行权限提升

// ❌ 禁止：通过 __proto__ 或 constructor 绕过沙箱
({}).__proto__.xxx = yyy; // 禁止！
```

## 附录 B：关键概念速查表

### 进程隔离

| 概念 | 说明 |
|------|------|
| **contextIsolation** | 隔离渲染进程 JavaScript 执行上下文，防止访问 preload 中的 Node.js API |
| **nodeIntegration: false** | 禁用渲染进程直接调用 Node.js API（如 require、process 等） |
| **sandbox: true** | 启用 Chromium 沙箱，额外限制渲染进程对 OS 资源的访问（文件、网络等） |
| **preload 脚本** | 在渲染进程加载前执行，通过 contextBridge 暴露受限 API 给业务代码 |

### 通信机制

| 概念 | 说明 |
|------|------|
| **Unix Socket / Named Pipe** | SkillApp 与 IntentOS Desktop 的进程间通信通道（非 HTTP） |
| **JSON-RPC 2.0** | 通信协议标准，支持请求-响应和单向通知 |
| **IPC Channel** | Electron IPC 的消息频道名称（如 `runtime:callSkill`） |

### 热更新

| 概念 | 说明 |
|------|------|
| **动态 import()** | 运行时加载 JavaScript 模块，支持替换已加载的模块 |
| **webContents.reloadIgnoringCache()** | 强制重载渲染进程，用于热更新失败的降级方案 |
| **模块级粒度** | 热更新的替换单位是整个文件/模块，不是函数级 |
